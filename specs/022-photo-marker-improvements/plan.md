# Implementation Plan: Photo Marker Improvements

**Branch**: `022-photo-marker-improvements` | **Date**: 2026-03-13 | **Spec**: [spec.md](spec.md)

## Summary

Fix photo marker jitter during map pan and reduce marker visual footprint. The jitter is caused by `ViewportSampler.update()` using screen-space pixel coordinates (`latLngToContainerPoint`) for cell assignment — these shift with every pan, causing mass marker replacement on every `moveend`. The fix is a one-line change to use geo-stable world-pixel coordinates (`map.project`) instead. Visual changes (smaller markers, thinner border, more default density) are pure CSS and TIER_CONFIG constant tweaks.

## Phase 1: Design

### Change 1 — Geo-stable cell key (ViewportSampler.js)

**Current** (screen-space, shifts with every pan):
```javascript
var pt = _map.latLngToContainerPoint(L.latLng(photo.lat, photo.lng));
var cellX = Math.floor(pt.x / _cellSize);
var cellY = Math.floor(pt.y / _cellSize);
var cellKey = cellX + ',' + cellY;
```

**Replace with** (world-pixel at current zoom, geo-stable across pans):
```javascript
var pt = _map.project(L.latLng(photo.lat, photo.lng), _map.getZoom());
var cellX = Math.floor(pt.x / _cellSize);
var cellY = Math.floor(pt.y / _cellSize);
var cellKey = cellX + ',' + cellY;
```

**Why this works**: `map.project(latlng, zoom)` returns the same world-pixel for a given lat/lng at a given zoom, regardless of the current pan position. Cell assignments are now stable across pans — the diff keeps all existing markers on `moveend`. Markers only re-cluster when zoom changes (which is expected behavior).

**Downstream note**: `ViewportSampler.update()` is also called by `rebuildPhotoLayer()` (on favorite toggle, timeline filter, auth change). These calls will also benefit from the geo-stable clustering — markers will be preserved where possible.

### Change 2 — Reduce TIER_CONFIG sizes (ViewportSampler.js)

**Current**:
```javascript
var TIER_CONFIG = [
    { maxCount: 1,        frameSize: 70,  stemHeight: 0  },
    { maxCount: 5,        frameSize: 85,  stemHeight: 12 },
    { maxCount: 15,       frameSize: 100, stemHeight: 14 },
    { maxCount: Infinity, frameSize: 115, stemHeight: 16 }
];
```

**Replace with** (~25% smaller across all tiers):
```javascript
var TIER_CONFIG = [
    { maxCount: 1,        frameSize: 52,  stemHeight: 0  },
    { maxCount: 5,        frameSize: 64,  stemHeight: 8  },
    { maxCount: 15,       frameSize: 76,  stemHeight: 10 },
    { maxCount: Infinity, frameSize: 88,  stemHeight: 12 }
];
```

### Change 3 — Reduce default cell size (ViewportSampler.js)

**Current**: `var _cellSize = 150;`
**Replace with**: `var _cellSize = 100;`

This allows ~2.25× more cells to be visible simultaneously at any zoom level, revealing the geographic distribution of photos within a city without requiring the user to adjust the density slider.

---

### Change 4 — Thinner photo frame border (Leaflet.Photo.css)

**Current**: `border: 3px solid white;`
**Replace with**: `border: 2px solid white;`
